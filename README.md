# Team5
大家取名时尽量规范些，根据英文单词和驼峰命名法
在每个脚本前使用<<summary></summary>>进行注释，标注这个脚本是用来做什么,例子如下：

  ////<summary>
  	  这是一个控制player移动的脚本
  ////</summary>


在较长的代码文件中，使用#region #endregion来折叠或隐藏一个或多个区域，这样，可将精力集中于当前处理的文件部分,便于大家分辨内容。 
演示如下：

 #region MyClass的类的定义  
public class MyClass   
{  
    static void Main()   
    {  
    }  
}  
#endregion
